# Требования SEO: лендинг расширения Steal the Vibe (3 кластера)

> Документ фиксирует требования к структуре сайта и страницам для охвата **трёх семантических кластеров** без программатического SEO (без массовой генерации URL из шаблонов).  
> Источник сигналов: Google Trends / см. `docs/wordstat/steal_the_vibe/`.  
> Связанные документы: `docs/extension-landing-pain-hope-solution.md` (маркетинговая структура), `docs/extension-ui-spec.md`.

---

## 1. Область и цели

### 1.1. Три кластера (EN, основной охват лендинга)

| Кластер | Смысл интента | Примечание |
|--------|----------------|------------|
| **AI photo prompt** | Пользователь ищет **текст промпта** под генерацию/редактирование фото (нейросеть). | Инструментальный + частично информационный. |
| **Image to prompt** | Явный запрос **«картинка → текст промпта»** (reverse prompt, описание стиля). | Самый близкий к продукту кластер. |
| **AI Photo Generator** | Ожидание **генерации фото** (часто «free», «from photo» в смежных запросах). | Коммерческий; продукт — **не замена** любого «generator», нужна честная посадка и мост к ценности. |

### 1.2. Вне скоупа этого документа

- Программатический SEO (тысячи страниц по шаблонам `[product]×[city]` и т.п.).
- Автогенерация текстов по спискам URL.
- Региональные клоны страниц (если понадобятся — отдельное ТЗ).

### 1.3. Цель документа

Зафиксировать:

- какую **структуру сайта** делаем под 3 кластера;
- **какие запросы на какие страницы** ведём (разводка);
- **достаточно ли объёма текста** и какого типа блоки нужны;
- **технические** требования к лендингу для индексации и сниппетов.

## 2. Структура сайта (минимальный набор страниц)

### 2.1. Рекомендуемая иерархия

Достаточно **одной основной посадочной** на продукт + **опционально** второй узкой страницы, если не хотим перегружать один URL тремя равными H1-интентами.

| Вариант | Страницы | Когда выбирать |
|--------|----------|----------------|
| **A (минимум)** | **1 URL** — продуктовый лендинг расширения (см. §3.1). | Быстрый старт, один canonical, проще поддержка. |
| **B (рекомендуется для SEO)** | **2 URL:** (1) главный продукт STV, (2) страница/раздел под **image to prompt** как отдельный интент. | Чёткое разведение «инструмент из картинки» vs «промпт + генератор в целом». |

**Имена путей (пример, финально согласовать с прод-хостом):**

- Основной лендинг: `/steal-the-vibe` или `/extension/steal-the-vibe` (короче и стабильнее для ссылок, чем технический `/extension-stv`).
- Второй URL (вариант B): `/image-to-prompt` **или** тот же путь с якорем `#image-to-prompt` — **якорь хуже для отдельного сниппета в выдаче**; для отдельного кластера предпочтительна **отдельная страница** с уникальным title/description.

**Связь с каталогом PromptShot (`/` и листинги):**

- С главной и 2–4 высокотрафиковых разделов — **внутренние ссылки** на лендинг расширения (см. §7).

### 2.2. Языки

- Для **РФ / Яндекс**: отдельная **RU-версия** с **уникальными** `title`, `description`, `H1` и основным текстом (не только автоперевод).
- Для **глобального** охвата кластеров в EN — базовая страница на английском.
- Связка: `hreflang` `ru` / `en` / `x-default`, если обе версии в проде.

## 3. Разводка запросов по страницам

### 3.1. Страница P1 — «Главный продукт» (Steal the Vibe / расширение)

**Назначение:** бренд + установка + ценность «на любом сайте», Pinterest, референс → промпт → свой снимок.

**Первичный кластер:** **AI photo prompt** (широкий: «промпт для фото ИИ»).

**Вторичные (в тексте и H2, не дублируя H1):** image to prompt, нейтральное упоминание AI photo generator как смежного интента.

**Примеры формулировок, которые приземляются на P1:**

- ai photo prompt / ai photo prompts  
- prompt for ai photo / photo prompt ai  
- (часть) «how to write prompt for ai photo» — через информационный блок + FAQ  

**Meta-логика:**

- **Title:** обязан содержать ядро **AI photo prompt** или близкую естественную форму + бренд (лимит ~60 символов, без спама).  
- **H1:** согласовать с брендом и стором (вариант 3 из `extension-landing-pain-hope-solution.md`) **или** отдельное правило «web H1 vs store title» — зафиксировать в релизе (см. §8).

### 3.2. Страница P2 — «Image to prompt» (вариант B)

**Назначение:** узкий интент «изображение → текст промпта», сравнение с ручным описанием, сценарии (Pinterest, stock, соцсети).

**Первичный кластер:** **image to prompt**.

**Примеры запросов на P2:**

- image to prompt  
- picture to prompt / photo to prompt (как синонимы в тексте)  
- reverse image prompt / prompt from image (если используете в копирайте — осторожно, не путать с «reverse image search»)

**Обязательно:** явный **H1** с формулировкой **Image to prompt** (или RU-эквивалент в зеркале).

**CTA:** та же установка расширения + кратко «тот же продукт, что Steal the Vibe».

### 3.3. Страница P3 — «AI Photo Generator» (опционально)

**Варианты:**

1. **Не выделять отдельный URL** — блок на P1 с H2 вроде «Not just an AI photo generator» + честное пояснение (промпт из референса + генерация у вас в экосистеме).  
2. **Отдельная короткая страница** только если есть **однозначный** оффер под «generator» (иначе — высокий отказ и жалобы в поведенческих).

Для текущего продукта (расширение = **извлечение промпта + стиль на своё фото**) **рекомендация:** **без отдельного P3**, кластер **AI Photo Generator** закрывать **секцией + FAQ на P1** (см. §5).

### 3.4. Сводная таблица «кластер → страница»

| Кластер | Основная посадка | Доп. усиление |
|--------|-------------------|---------------|
| AI photo prompt | **P1** | H2, первый экран, FAQ |
| Image to prompt | **P2** (или P1 + H2 `#…` — хуже для SEO) | Уникальный title/description на P2 |
| AI Photo Generator | **P1** (секция + FAQ) | При появлении полноценного генератора — пересмотреть P3 |

## 4. Объём и типы текста (хватает ли контента)

### 4.1. Минимальные ориентиры (без программатика)

| Страница | Минимум основного текста (видимый пользователю) | Заметки |
|----------|-----------------------------------------------|---------|
| P1 | **400–800 слов** суммарно по секциям (герой + боль + решение + доверие) | Сейчас превью — в основном UI-мокапы; для SEO нужны **абзацы** под H2. |
| P2 | **300–600 слов** | Акцент на «что такое image to prompt», шаги, ограничения. |

### 4.2. Обязательные блоки (обе страницы при варианте B)

1. **Один внятный вводный абзац** (50–90 слов) с главным ключом страницы.  
2. **2–4 H2** с **разными** формулировками (охват LSI, без набивки).  
3. **FAQ 4–6 вопросов** с ответами по 40–80 слов — закрывает long-tail и даёт разметку FAQPage.  
4. **Короткий disclaimer** (модель, качество исходника) — доверие и E-E-A-T.  

### 4.3. Оценка текущего превью `/extension-stv`

- **Недостаточно** для конкурентного SEO по трём кластерам: преобладают короткие фразы и мокапы.  
- Для запуска SEO-режима: **добавить** SEO-блоки (§4.2) и **не** полагаться только на subtitle из стора.

## 5. Структура SEO-текста по страницам (черновик)

### 5.1. P1 — главный лендинг

| Элемент | Требование |
|---------|------------|
| Title | Ключ **AI photo prompt** (или естественная форма) + бренд; уникальный для RU/EN. |
| Meta description | **Image to prompt** и/или recreate / style; 150–160 символов; без обмана про «free unlimited generator». |
| H1 | По согласованию: store-align или web-оптимизация (зафиксировать). |
| H2 (примеры) | «AI photo prompt from any reference», «Image to prompt in your browser», «How we’re different from a generic AI photo generator». |
| FAQ | В т.ч. вопрос: «Is this an AI photo generator?» — честный ответ + ценность промпта. |

### 5.2. P2 — image to prompt

| Элемент | Требование |
|---------|------------|
| Title | Начинается или содержит **Image to prompt** + бренд/уточнение. |
| H1 | **Image to prompt** (или RU: «Промпт из картинки» / «Картинка в промпт»). |
| Контент | Сценарии: Pinterest, moodboard, рекламные креативы; 3–5 шагов; отличие от ручного описания. |
| Внутренняя ссылка | На P1 как «полный продукт Steal the Vibe». |

## 6. Технические требования к лендингу

### 6.1. Индексация и каноникал

- Снять **`noindex`** на прод-лендинге (сейчас у превью `/extension-stv` — черновик).  
- Явный **`canonical`** на выбранный постоянный URL (без дублей с query-параметрами).  
- **Не** отдавать одинаковый контент на разных путях без canonical.

### 6.2. Разметка

- **JSON-LD:** `SoftwareApplication` / `WebApplication` (название, описание, `offers` при необходимости, ссылка на Chrome Web Store).  
- **FAQPage** — для блока FAQ на P1 (и P2 при наличии).  
- **Organization** (опционально) — связь с PromptShot.

### 6.3. Производительность и поведение (Яндекс / Google)

- LCP hero: не перегружать первый экран тяжёлыми изображениями; приоритет тексту и одному ключевому визуалу.  
- Стабильный layout (CLS): плавающая CTA не должна перекрывать читаемый текст при скролле (отступы `padding-bottom` у контента).  

### 6.4. hreflang

- При публикации RU + EN: теги `link rel="alternate" hreflang` + `x-default`.

### 6.5. Sitemap

- Включить P1 и P2 в `sitemap.xml` после снятия noindex.

## 7. Внутренняя перелинковка (минимум)

- С **главной** сайта — ссылка на P1 анкором с ротацией: «расширение», «image to prompt», «промпт по картинке» (не все с одной страницы одинаково — естественность).  
- С **2–4 популярных категорий** каталога — ссылка на P1 или P2 по релевантности.  
- С P2 → P1; с P1 → P2 (текстовая ссылка в блоке «узнайте больше»).  
- Футер лендинга: ссылка на каталог, privacy, store.

---

## 8. Согласования перед внедрением

1. **Финальный URL** P1 (переименование с `/extension-stv` или редирект 301).  
2. **H1 vs Chrome Web Store title:** один документ «web vs store» — чтобы не ломать узнаваемость в сторе и не терять ключ на сайте.  
3. **Язык первой версии:** EN-only на старте или сразу RU+EN.  
4. **Нужна ли P2** с первого релиза SEO или этап 2.

---

## 9. Чеклист приёмки (кратко)

- [ ] P1 в проде с **index, follow**, корректный **canonical**.  
- [ ] Title/description P1 закрывают **AI photo prompt** + упоминание **image to prompt** (в description или H2).  
- [ ] На P1 есть **≥400 слов** полезного текста + **FAQ** + **JSON-LD FAQ + SoftwareApplication**.  
- [ ] Либо выпущена **P2** под **image to prompt** с уникальными meta и **≥300 слов**, либо зафиксировано осознанное решение отказа с рисками.  
- [ ] Кластер **AI Photo Generator** закрыт **честной** секцией на P1 (без ложного «free generator» без оффера).  
- [ ] Внутренние ссылки с главной и разделов каталога на P1/P2.  
- [ ] RU-зеркало и **hreflang** — если запускаем РФ.
