# SEO-требования: страницы карточек `/p/[slug]`

> Дата: 12 марта 2026
>
> Этот документ описывает SEO- и UI-требования только для страниц отдельных карточек промтов.
> Общая SEO-архитектура, теговые страницы, programmatic SEO описаны в `docs/07-03-prompt-landing-plan.md`.

---

## 1. Роль карточки в SEO-архитектуре

Карточка `/p/[slug]` — это **конечная целевая страница** под конкретный промт и конкретный визуальный результат.

В иерархии сайта:

```
Главная → Tag page (кластер) → Card page (конкретный промт)
```

Карточка выполняет 4 SEO-функции:

1. **Long-tail трафик.** Захватывает НЧ и ultra long-tail запросы, которые не покрываются tag pages.
2. **Перелинковка.** Связывает tag pages между собой через related-блоки и кликабельные теги.
3. **Поведенческие.** Дает четкий ответ на интент — фото, промт, копирование — что критично для Яндекса.
4. **Глубина.** Обеспечивает дополнительные переходы: related cards, group siblings, tag links.

Карточка **не конкурирует** с tag page за один и тот же кластерный запрос. Tag page закрывает кластер, карточка закрывает конкретный вариант внутри кластера.

## 2. Условия существования SEO-страницы

URL `/p/[slug]` существует как SEO-страница только если одновременно:

- `prompt_cards.is_published = true`
- `prompt_cards.slug IS NOT NULL`

Если хотя бы одно условие не выполнено:

- URL не попадает в sitemap;
- URL не участвует во внутренней перелинковке (related, group siblings, breadcrumb);
- страница отдает 404 или noindex.

### Автоматическое появление

Публикация карточки в БД автоматически приводит к:

1. появлению рабочего URL `/p/[slug]`;
2. записи в `sitemap.xml`;
3. участию карточки в блоках related и group navigation на других страницах.

Никаких ручных действий для добавления URL в структуру сайта не требуется.

## 3. Sitemap

### Требования

- в sitemap попадают только карточки с `is_published = true` и `slug IS NOT NULL`;
- новые карточки появляются автоматически после публикации;
- при снятии с публикации карточка исчезает из sitemap;
- `lastModified` берется из `updated_at` в БД, а не `new Date()`;
- URL: `https://promptshot.ru/p/{slug}`.

### Проектные особенности

Текущий `sitemap.ts` использует `getPublishedCardSlugs()` и `lastModified: new Date()`.
Нужно заменить на запрос с реальной датой:

```sql
SELECT slug, updated_at
FROM prompt_cards
WHERE is_published = true AND slug IS NOT NULL
```

При росте карточек > 5000 — перейти на `generateSitemaps()` (sitemap index) с разделением на `sitemap-tags.xml` и `sitemap-cards-{id}.xml`.

## 4. Metadata

### Обязательные поля

Для каждой `/p/[slug]` генерируются уникальные:

- `title`
- `description`
- `canonical`
- `openGraph`
- `twitter`
- `robots: index, follow`

### Шаблоны

**Title** (до 60 символов, Яндекс обрезает длиннее):

```
{title_ru} — промт для фото ИИ | PromptShot
```

Если `title_ru` длинный — обрезать title_ru, не трогать суффикс.

**Description** (до 160 символов):

Собирается из доступных данных по приоритету:

1. Если есть промт: `Промт для фото: «{первые 100 символов промта}…». Скопируй и создай фото в нейросети.`
2. Если есть теги но нет промта: `Готовый промт «{title}» — {tag1}, {tag2}. Копируй и используй в ИИ.`
3. Fallback: `Готовый промт для генерации фото ИИ. Посмотри результат и скопируй.`

**Canonical:**

```
https://promptshot.ru/p/{slug}
```

**OpenGraph / Twitter:**

- `og:image` — главное фото карточки (первое из `photoUrls`);
- `og:type` — `article`;
- `twitter:card` — `summary_large_image`.

### Проектные особенности

Сейчас на `/p/[slug]` нет `generateMetadata` — все карточки используют дефолтный title из `layout.tsx`. Нужно добавить `generateMetadata` в `page.tsx`, используя данные, которые уже загружаются в `CardPage`.

Важно для Яндекса: title должен быть уникален для каждой карточки. Яндекс агрессивно пессимизирует шаблонные title, которые отличаются только одним словом. Поэтому title строится из `title_ru`, который уже уникален для каждой карточки.

## 5. Structured Data (JSON-LD)

Тип: `CreativeWork`.

```json
{
  "@context": "https://schema.org",
  "@type": "CreativeWork",
  "name": "{title_ru}",
  "description": "{первые 150 символов промта}",
  "image": "{main_photo_url}",
  "url": "https://promptshot.ru/p/{slug}",
  "datePublished": "{source_date}",
  "keywords": "{tag1}, {tag2}, {tag3}",
  "isPartOf": {
    "@type": "CollectionPage",
    "name": "PromptShot — промты для фото ИИ",
    "url": "https://promptshot.ru"
  }
}
```

### Дополнительно, если есть before/after

Если у карточки есть `beforePhotoUrl`, добавить `ImageObject` с двумя изображениями. Это усиливает visual rich snippets в Google.

### Breadcrumb JSON-LD

Отдельный блок `BreadcrumbList`:

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Главная", "item": "https://promptshot.ru" },
    { "@type": "ListItem", "position": 2, "name": "{tag_label}", "item": "https://promptshot.ru{tag_url_path}" },
    { "@type": "ListItem", "position": 3, "name": "{title_ru}" }
  ]
}
```

Для breadcrumb берется **первый** тег из `seo_tags` по приоритету: `audience_tag` > `style_tag` > `occasion_tag` > `object_tag` > `doc_task_tag`.

## 6. UI страницы карточки

### Принцип

Пользователь пришел по запросу «промт для фото девушки в черно-белом стиле». Он должен за 2 секунды увидеть результат, за 5 секунд скопировать промт, за 10 секунд перейти к похожим вариантам.

Это критично для поведенческих факторов Яндекса: быстрый ответ на интент → низкий bounce → высокая глубина.

### Структура страницы (сверху вниз)

```
┌──────────────────────────────────────────────────┐
│  1. Hero: основное фото                          │
│     - бейдж БЫЛО (top-left), если есть before    │
│     - счетчик фото (top-right), если > 1         │
│     - thumbnail strip, если > 1 фото             │
├──────────────────────────────────────────────────┤
│  2. Breadcrumb: Главная → Tag page → Карточка    │
│                                                  │
│  3. H1: {title_ru}                               │
│                                                  │
│  4. Теги (кликабельные → tag pages)              │
│                                                  │
│  5. Промт                                        │
│     - полный текст в читаемом блоке              │
│     - кнопка «Скопировать промт»                 │
│                                                  │
│  6. Было / Стало (если есть beforePhotoUrl)      │
│     - две фотографии рядом, полноразмерные       │
│                                                  │
│  7. Группа (если cardSplitTotal > 1)             │
│     - «Промт 2 из 3»                            │
│     - превью + ссылки на соседние карточки        │
│                                                  │
│  8. Похожие промты                               │
│     - 4–6 карточек по основному тегу             │
│     - ссылка «Все промты для {tag}» → tag page   │
│                                                  │
│  9. Footer                                       │
└──────────────────────────────────────────────────┘
```

### Требования к каждому блоку

**Hero:** above the fold — фото и заголовок. Пользователь сразу понимает, какой результат дает этот промт.

**Breadcrumb:** видимый и кликабельный. Для Яндекса breadcrumb — сигнал структуры сайта. Берем первый тег из `seo_tags` как промежуточный уровень.

**H1:** совпадает с `title_ru`. Один H1 на страницу.

**Теги:** каждый тег — ссылка на соответствующую tag page из `TAG_REGISTRY`. Теги, которых нет в реестре, показываются без ссылки. Это дает:
- перелинковку card → tag page;
- сигнал релевантности для поисковика;
- следующий клик для пользователя.

**Промт:** полный текст, не превью. Блок должен быть визуально выделен (фон, рамка). Кнопка копирования — заметная, с фидбеком «Скопировано!».

**Было / Стало:** если есть `beforePhotoUrl`, показывать отдельную секцию с двумя фото рядом. Это:
- увеличивает время на странице;
- усиливает CTR из выдачи (если фото попадает в visual snippets);
- дает дополнительный контент для индексации.

**Похожие промты:** 4–6 карточек, отсортированных по `seo_readiness_score`. Обязательна ссылка «Все промты для {tag}» — это перелинковка card → tag page снизу страницы.

## 7. Группы

### Определение

Группа — это несколько карточек из одного исходного поста (`source_dataset_slug` + `source_message_id`, `card_split_total > 1`).

### SEO-правила

- каждая карточка группы — самостоятельная SEO-страница со своим `slug`, своим canonical, своим title;
- группа **не** создает отдельный SEO-роут;
- группа влияет только на UI и перелинковку;
- каждая карточка группы индексируется самостоятельно.

### На странице карточки

- показывать метку `Промт {N} из {total}`;
- показывать превью соседних карточек с ссылками на их `/p/[slug]`;
- блок называется «Другие промты из этого поста».

### SEO-связь

Добавить в `<head>`:

```html
<link rel="prev" href="/p/{prev_slug}" />
<link rel="next" href="/p/{next_slug}" />
```

Только между соседними по `card_split_index`. Не добавлять, если карточка первая/последняя.

### Загрузка соседей

```typescript
if (card.source_dataset_slug && card.source_message_id && card.card_split_total > 1) {
  const siblings = await supabase
    .from("prompt_cards")
    .select("id,slug,title_ru,card_split_index")
    .eq("source_dataset_slug", card.source_dataset_slug)
    .eq("source_message_id", card.source_message_id)
    .eq("is_published", true)
    .not("slug", "is", null)
    .order("card_split_index", { ascending: true });
}
```

## 8. Внутренняя перелинковка

Карточка не должна быть тупиковой страницей. Каждая карточка должна содержать ссылки:

### Вверх по иерархии

- breadcrumb → tag page → главная;
- кликабельные теги → соответствующие tag pages.

### Горизонтально

- похожие карточки (по тегам);
- соседние карточки группы.

### Вниз (на карточку)

Карточка должна быть доступна:
- из tag page (как элемент сетки);
- из блока related на других карточках;
- из блока group siblings;
- из sitemap.

### Проектные особенности

Сейчас tag pages уже рендерят карточки через `FilterableGrid`. При клике на карточку пользователь должен переходить на `/p/[slug]`. Обратная ссылка — через breadcrumb и кликабельные теги.

Для перелинковки между карточками нужен запрос похожих карточек по тегам. Минимальная логика:

```sql
SELECT id, slug, title_ru
FROM prompt_cards
WHERE is_published = true AND slug IS NOT NULL
  AND id != {current_card_id}
  AND seo_tags->'audience_tag' @> '["devushka"]'
ORDER BY seo_readiness_score DESC
LIMIT 6
```

## 9. Оптимизация под Яндекс

### Поведенческие факторы

Страница карточки оптимизирована для быстрого удовлетворения интента:
- фото above the fold;
- промт виден без скролла или с минимальным скроллом;
- копирование — одним кликом;
- следующий переход очевиден (related, tags, group).

### Уникальность контента

- каждая карточка имеет уникальный title (из `title_ru`);
- description собирается из реального текста промта;
- промт — это уникальный пользовательский контент, не шаблон;
- фото — уникальные результаты генерации.

### Thin content

Карточки без промта или без фото не должны индексироваться. Добавить условие:

- если `promptTexts.length === 0` и `photoUrls.length === 0` → `robots: noindex, follow`.

### Regional

Сайт ориентирован на Россию/СНГ. Контент на русском. Яндекс Вебмастер должен быть настроен на регион «Россия».

## 10. Текущее состояние и gap list

| Компонент | Статус | Задача |
|-----------|--------|--------|
| Роут `/p/[slug]` | Есть | Оставить |
| Загрузка по slug | Есть | Оставить |
| `generateMetadata` | **Нет** | Добавить уникальные title, description, OG, canonical |
| JSON-LD | **Нет** | Добавить `CreativeWork` + `BreadcrumbList` |
| Canonical | **Нет** | Добавить `alternates.canonical` |
| OG / Twitter | **Нет** | Добавить с главным фото |
| Sitemap `lastModified` | **Нет** (всегда `new Date()`) | Брать `updated_at` из БД |
| Кликабельные теги | **Нет** | Связать с `TAG_REGISTRY` → ссылки на tag pages |
| Breadcrumb | Есть, но без тега | Добавить промежуточный уровень (tag page) |
| Блок группы | **Нет** | Добавить навигацию по siblings |
| Блок related | **Нет** | Добавить похожие карточки по тегам |
| Секция «Было / Стало» | Частично (только бейдж) | Добавить полноразмерную секцию |
| `rel="prev/next"` | **Нет** | Добавить для групп |
| Noindex для thin content | **Нет** | Добавить проверку на пустой промт + пустые фото |
| Переход из tag page в card page | **Нет** | Карточки в сетке должны вести на `/p/[slug]` |
