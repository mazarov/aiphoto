# ТЗ: SEO-посадочные страницы для photo2sticker.ru

**Дата:** 2026-02-11
**Проект:** photo2sticker.ru — лендинг для Telegram-бота создания стикеров из фото
**Цель:** Создание системы посадочных страниц для привлечения органического трафика из Яндекса и Google

---

## 1. Контекст и текущее состояние

### 1.1. Продукт
Telegram-бот [@Photo_2_StickerBot](https://t.me/Photo_2_StickerBot) — превращает фото в стикеры с помощью ИИ (Gemini). Поддерживает:
- 10 групп стилей, ~45 подстилей (аниме, мемы, милые, романтика, мультфильм, игровой, рисунок, манхва, сериалы, русский стиль)
- Эмоции (смех, грусть, злость и т.д.)
- Анимацию/движение (машет рукой, танцует и т.д.)
- Добавление текста на стикер
- Удаление фона (rembg + Pixian)

### 1.2. Текущий сайт
- **Домен:** https://photo2sticker.ru/
- **Стек:** Vite + React + Tailwind CSS + shadcn/ui
- **Деплой:** Vercel
- **Состояние:** Одностраничный SPA (client-side rendering)
- **Исходники:** `/Users/azarovmaxim/photo2sticker/`

### 1.3. Критическая проблема
Сайт — **SPA с client-side рендерингом**. В HTML отдаётся пустой `<div id="root"></div>`, контент рендерится JS в браузере. Яндекс плохо индексирует JS-контент. Все новые страницы будут невидимы для поисковиков без решения этой проблемы.

**Решение:** Переход на SSG (Static Site Generation) или SSR. Рекомендация — Vite SSG для первой фазы с возможной миграцией на Next.js позже.

### 1.4. Текущая структура сайта

```

│   ├── index.html                    # Единственный HTML, хардкод title/meta
```

## 6. Техническая реализация

### 6.1. Фаза 0: Миграция на Next.js App Router (критический путь)

**Задача:** Сделать так, чтобы каждый URL отдавал уникальный HTML с title, meta, h1, контентом — без JS-рендеринга.

**Анализ вариантов:**

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Vite SSG** (vite-ssg) | Минимальный рефактор текущего кода | Нет ISR, плохо поддерживается, нет API routes, временное решение |
| **Astro** | SSG по умолчанию, быстрый | Другая парадигма (MPA), нет нативного ISR на Vercel, каждый интерактивный компонент — island |
| **Next.js App Router** | SSR + ISR + SSG + API routes + Server Components + нативный Vercel | Полная миграция (но текущий код маленький) |

**Решение: Next.js App Router.**

Причины:
1. **Vercel-нативный** — нулевой конфиг, лучшая производительность на этом хостинге
2. **`generateStaticParams()`** — генерит 65+ HTML-файлов при билде, идеально для SEO
3. **ISR (Incremental Static Regeneration)** — галереи обновляются без ребилда всего сайта
4. **Server Components** — запрос к Supabase прямо в компоненте, без отдельного API
5. **`metadata` API** — уникальный title/description/canonical/OG на каждой странице нативно
6. **`next/image`** — автоматический WebP, lazy loading, CDN-оптимизация
7. **Текущий стек (React + Tailwind + shadcn/ui)** — переносится 1:1 без изменений
8. **Текущий сайт маленький** (1 страница, ~10 компонентов) — миграция за 2-3 дня

**Структура после миграции:**
```
app/
├── layout.tsx                         ← общий layout (шрифты, Яндекс.Метрика)
├── page.tsx                           ← главная /
├── stiker-iz-foto/page.tsx            ← кластерная
├── bot/page.tsx                       ← кластерная
├── telegram/page.tsx                  ← кластерная
├── iphone/page.tsx                    ← кластерная
├── android/page.tsx                   ← кластерная
├── animirovannye/page.tsx             ← кластерная
├── s-nadpisyu/page.tsx                ← кластерная
├── besplatno/page.tsx                 ← кластерная
├── style/
│   ├── page.tsx                       ← каталог стилей /style/
│   ├── [group]/
│   │   ├── page.tsx                   ← группа /style/anime (generateStaticParams)
│   │   └── [substyle]/
│   │       └── page.tsx               ← подстиль /style/anime/chibi (generateStaticParams)
├── api/

```

**Что переносится без изменений:**
- Все React-компоненты (`Hero.tsx`, `FAQ.tsx`, `StyleGallery.tsx` и т.д.)

**Что меняется:**

### 6.4. Фаза 3: Хранение изображений для сайта

**Проблема:** Стикеры хранятся как `telegram_file_id` — это внутренний ID Telegram, не публичный URL. Для сайта нужны HTTP-ссылки на картинки.

**Решение: Supabase Storage**

1. **Новая колонка в `stickers`:**
```sql
ALTER TABLE stickers ADD COLUMN IF NOT EXISTS public_url text;
```

2. **Изменения в `src/worker.ts`:**
При генерации стикера, после получения PNG:
```typescript
// Upload to Supabase Storage
const { data } = await supabase.storage
  .from('sticker-examples')
  .upload(`${stickerId}.webp`, webpBuffer, {
    contentType: 'image/webp',
    upsert: true,
  });

// Save public URL
const publicUrl = supabase.storage
  .from('sticker-examples')
  .getPublicUrl(`${stickerId}.webp`).data.publicUrl;

await supabase.from('stickers')
  .update({ public_url: publicUrl })
  .eq('id', stickerId);
```

3. **API для сайта:**
```
GET /api/styles/{groupId}/examples?limit=16
→ [{ id, public_url, style_preset_id, created_at }]
```

4. **Когда сохранять:**
- Вариант A: Сохранять ВСЕ стикеры (дорого по хранилищу)
- Вариант B: Сохранять только `is_example = true` (рекомендуется)
- Вариант C: Сохранять все, но на сайте показывать только `is_example = true`

**Рекомендация:** Вариант B — экономим storage, показываем только курированный контент.

## 7. Антидубль-стратегия

Каждая страница должна быть **существенно уникальной**, а не перефразированной главной.

### 7.1. Обязательные уникальные элементы

| Элемент | Правило |
|---------|---------|
| `<title>` | 100% уникальный, содержит целевой запрос |
| `<meta description>` | 100% уникальный, ~155 символов |
| `<h1>` | 100% уникальный, отличается от title |
| SEO-текст | 100% уникальный, 400-600 слов |
| FAQ | 100% уникальный набор вопросов из Wordstat |
| Изображения | Разные примеры стикеров на каждой странице |
| `<canonical>` | Указывает на саму себя (не на главную!) |

### 7.2. Переиспользуемые компоненты (допустимо)

- Шапка/футер — одинаковые (нормально)
- Структура "Как это работает" — шаблон одинаковый, **текст шагов разный**
- CTA-кнопка — одинаковый дизайн, **разный текст и UTM**
- TrustBar — одинаковый (нормально для UI-элемента)

### 7.3. Что делает страницу НЕ дублем

Кластерная `/iphone` — это не "главная + слово iPhone". Это:
- **Сравнительная статья:** 3 способа сделать стикер на iPhone (встроенный iOS Live Photo, приложения, наш бот)
- **Скриншоты:** реальные скрины на iPhone

- **Специфичные инструкции:** "Откройте Telegram на iPhone → найдите бота..."

Стилевая `/style/anime` — это не "главная в стиле аниме". Это:
- **Каталог подстилей** с превью и описаниями
- **Галерея реальных примеров** этого стиля
