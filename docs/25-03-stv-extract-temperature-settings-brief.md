# Бриф: temperature для `/api/vibe/extract` — UI в блоке «Настройки»

> **Версия:** 2026-03-22  
> **Статус:** реализовано (extension «Настройки» + `POST /api/vibe/extract` body `extractTemperature`, clamp 0–2).

## Цель

Дать пользователю **явный контроль над `temperature`** при извлечении стиля из референса (legacy extract), чтобы балансировать **стабильность JSON** (низкая температура) и **вариативность формулировок** (выше).

## Где в UI (только «Настройки»)

- **Размещение:** секция **«Настройки»** (в extension — блок со step **2**, заголовок `section_settings` в `extension/sidepanel/app.js`), **после** или **перед** существующими полями модели / aspect ratio / image size — визуально одна группа с остальными параметрами генерации.
- **Не размещать** в блоке **«Внешний вид (референс)»** (`grooming_title` / `stv-grooming-block--main`) — там только политика волос/макияжа и связанные подсказки.

### Рекомендуемый контрол

- **Range slider** `0.0` … `1.0` (шаг `0.05`) **или** компактный **select** с пресетами: `Точнее (0.1)` / `Сбалансированно (0.3)` / `Свободнее (0.6)` + опционально «По умолчанию API».
- Подпись и короткий hint: что влияет **только на extract** (анализ референса), не на image-gen модель из соседнего селекта.

## Поведение и данные

| Аспект | Решение |
|--------|---------|
| **Значение по умолчанию** | «По умолчанию API» = **не передавать** `temperature` в запросе (как сейчас), либо явный дефолт **0.2** для extract — зафиксировать в реализации одним способом. |
| **Персистенция** | Сохранять в **`chrome.storage.local`** вместе с `stv_state_v2` (новое поле, например `extractTemperature: number \| null`) или отдельный ключ; при `null` — режим «API default». |
| **Когда применяется** | Только при **`POST /api/vibe/extract`** в `runExtract()`. |

## API

- **Запрос:** опциональное поле в JSON body, например `extractTemperature?: number \| null`.
- **Сервер (`landing/.../vibe/extract/route.ts`):**
  - Если поле **отсутствует** или `null` — не задавать `temperature` в Gemini / OpenAI (текущее поведение).
  - Если число — **clamp** на сервере: например `[0, 2]` для OpenAI и `[0, 2]` для Gemini (уточнить по актуальной доке модели; минимум **0** и **верхняя граница** жёстко в коде).
  - **Не доверять** клиенту: игнорировать отрицательные и NaN; при невалидном значении — 400 или fallback на default.
- **Gemini:** добавить в `generationConfig` поле `temperature`, если значение задано.
- **OpenAI:** передать `temperature` в body `chat/completions` в `openAiExtractImageJson` / общий helper (сейчас параметр не передаётся).

## Логи и отладка

- В `console.warn` для extract (уже есть `gemini_request` / `openai_request`) добавить поле **`extractTemperature`** (число или `"default"`).

## Вне скоупа

- Temperature для **expand** (сейчас без LLM) или для **generate-process** / image model.
- Глобальный флаг в `photo_app_config` для дефолта (можно отложить; первый этап — только UI + body).

## Критерии готовности

1. Контрол виден **только** в секции «Настройки», не в «Внешний вид».  
2. Повторный extract с тем же референсом при смене слайдера даёт **различимую** (или задокументированно стабильную) вариативность в `style.camera` / формулировках.  
3. Сервер не принимает выход за допустимый диапазон.  
4. Состояние переживает перезапуск панели (persist).
