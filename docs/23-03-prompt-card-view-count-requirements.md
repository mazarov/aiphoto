# Требования: счётчик просмотров карточек промтов и ранжирование

> Дата: 23 марта 2026  
> Статус: в реализации (инкремент — миграция `sql/155_*` + `POST /api/card-view` + `useCardViewBeacon` на `/p/[slug]`; колонка и сортировки — `sql/154_*`)  
> Обновление: закрыты §7; ранжирование только по `view_count`; метрики в правом верхнем углу превью; правки expanded «Скопировать».

Документ задаёт **просмотр страницы карточки** `/p/[slug]` как метрику популярности, хранение в БД, участие в сортировке листингов и **отображение на UI карточки** (иконка + число).

Продуктовый/UI блок — ясность, минимализм, иерархия.  
Архитектурный блок и NFR — масштаб, отказоустойчивость, безопасность, эволюция.

---

## 1. Понимание продукта

**Продукт:** каталог промтов с визуальными примерами; пользователь ищет идею, открывает карточку, копирует промт или запускает генерацию.

**Проблема:** порядок карточек в сетках и на главной без явного сигнала «что реально смотрят люди».

**Решение:** **количество заходов на страницу карточки** — для ранжирования и лёгкого социального доказательства на превью (без перегруза интерфейса).

---

## 2. Определения

| Термин | Значение |
|--------|----------|
| **Просмотр карточки** | Зафиксированное событие: пользователь **впервые в рамках сессии браузера** открыл `/p/[slug]` опубликованной карточки и клиент отправил beacon на сервер. |
| **Сессия браузера** | Дедупликация через `sessionStorage`, ключ вида `promptshot_view_{slug}` или по `card_id`. |
| **Отображаемое число** | `view_count` из БД; на превью в сетке **всегда** показывать (при 0 — приглушённо). От 1000 — `formatCompactCount`. |

**Не считается просмотром:** SSR/RSC без клиентского beacon; повтор в той же сессии при уже выставленном ключе `sessionStorage` (рекомендуется строгий один просмотр на slug за сессию вкладки).

## 3. Функциональные требования

### 3.1 Данные

- `prompt_cards.view_count` — `bigint NOT NULL DEFAULT 0` (миграция **`sql/154_prompt_cards_view_count_ranking.sql`**).
- Индекс под сортировку: см. миграцию.

### 3.2 Инкремент

- Клиентский beacon после гидрации на `/p/[slug]` → `POST /api/card-view` → RPC `increment_prompt_card_view(p_slug)` (миграция **`sql/155_increment_prompt_card_view.sql`**).
- Сервер: атомарный `UPDATE ... SET view_count = view_count + 1` только для `is_published`.

### 3.3 Ранжирование (решение 7.1)

**Только по просмотрам:** порядок карточек в ленте/сетке и связанных RPC определяется **исключительно** как:

`ORDER BY view_count DESC NULLS LAST, source_date DESC NULLS LAST, id ASC`

(стабильный тай-брейк по дате и id).

**Не использовать** для сортировки листингов: `relevance_score`, `seo_readiness_score`, веса совпадения тегов. Поле `relevance_score` в JSON ответа `resolve_route_cards` может сохраняться для отладки/аналитики, **но не участвует в ORDER BY**.

**Поиск по тексту** (`search_cards_text`): релевантность по запросу **не менять**; опционально позже — `view_count` только как тай-брейк при равной релевантности.

**Реализация в БД (v1):** обновлены `resolve_route_cards`, `search_cards_filtered`, `get_homepage_sections` в миграции **154**.

### 3.4 Консистентность

- Single source of truth: `view_count` в БД; UI читает через enrich / API.

## 4. UI / UX (сетка карточек)

### 4.0 Дизайн-система лендинга

Согласовано с `GroupedCard` / `PromptCard`: `rounded-2xl`, градиент снизу, бейджи `rounded-full` + `bg-black/40` / `black/35` + `backdrop-blur-md`, `text-[10px]`, `tabular-nums`, акцент индиго только у чипа группы.

### 4.1 Размещение метрик на превью (решение 7.2 + вёрстка)

- **Просмотры, лайк, дизлайк** — **всегда видны**, без hover.
- Зона: **верхний правый угол** — колонка **`flex flex-col items-end gap-2`** без общей подложки/бордера; отступ `sm:right-3.5 sm:top-3.5`.
- Сверху вниз: счётчик фото (pill `bg-black/40 backdrop-blur`) → просмотры (отдельный pill) → лайк/дизлайк (`stacked`) → избранное (`PromptCard`). **Без** `border-t` между блоками.
- Просмотры **всегда**; при 0 — приглушённый pill; `aria-label` «Пока нет просмотров» при 0.
- **`PromptCard`:** в той же колонке ниже реакций — **избранное** (`FavoriteButton`), тоже без hover-скрытия.

Нижний градиент **не** содержит реакций — только **заголовок (1 строка)**, **превью промта (до 2 строк)** и кнопка **«Скопировать»**.

### 4.2 Просмотры — внешний вид

- Отдельный pill: `rounded-full backdrop-blur-md`, при `view_count > 0` — `bg-black/40 text-white/90`; при 0 — `bg-black/30 text-white/50`.
- Иконка глаз 14px; `aria-label` полная фраза.

### 4.3 Режим «Скопировать» (expanded)

- Полноэкранный оверлей **без** `onClick` на корневом контейнере (не блокировать кнопки и не ломать первый клик по копированию).
- **Теги убрать** из оверлея.
- Заголовок в шапке: **одна строка** (`line-clamp-1`) — первая часть из `splitCardTitle` (перенос строки в тексте или усечение по длине).
- Текст промта — в прокручиваемом блоке.
- **Остаток заголовка** (`rest` из `splitCardTitle`) — **под блоком промта**, мелким приглушённым текстом (`text-white/50`), если не пустой.
- Кнопка копирования: при успехе сразу показывать **«Промпт скопирован»** — `setCopied(true)` **до** `await navigator.clipboard.writeText`, при ошибке clipboard сбросить состояние.

### 4.4 Страница `/p/[slug]`

- Счётчик на `/p/[slug]` — спокойная строка под H1 (`text-sm text-zinc-500`); при желании скрывать при 0 (опционально, не обязательно как в сетке).

## 5. Архитектура и NFR (кратко)

- Запись просмотров: короткий POST, rate limit, best-effort; сбой не ломает страницу карточки.
- Hot row на `UPDATE` одной карточки — см. эволюцию в очереди/Redis при росте нагрузки.
- Метрики endpoint: RED-стиль, алерты на 5xx.

## 6. Критерии приёмки

- [ ] Миграция 154 применена; листинги сортируются по `view_count` (при равенстве — `source_date`, `id`).
- [ ] В сетке всегда видна метка просмотров (0 — приглушённо); действия без общей подложки и без разделителей между ними.
- [ ] Просмотры / лайк / дизлайк в **верхнем правом углу**, колонка, без hover; нижний блок — только заголовок (1 строка), превью промта (2 строки), «Скопировать».
- [ ] В expanded нет тегов; заголовок из двух частей по правилам `splitCardTitle`; кнопка копирования меняет текст с **первого** успешного нажатия.
- [x] Beacon просмотра на `/p/[slug]` и `POST /api/card-view` + `docs/architecture/01-landing.md`.
