# ТЗ: Локальный MVP фронта для просмотра карточек промтов

**Дата:** 11.03.2026  
**Проект:** aiphoto  
**Статус:** ready for implementation  
**Режим:** локальный прототип (без SEO, без прод-деплоя)

---

## 1. Цель

Сделать локальный одностраничный прототип, чтобы визуально оценить:

- как выглядит сетка карточек промтов;
- удобно ли фильтровать карточки по источнику данных (dataset/provider);
- достаточно ли данных в карточке для ручной ревизии и дальнейшего проектирования сайта.

Прототип нужен как UI-слой просмотра данных, а не как production-продукт.

---

## 2. Scope MVP

### Входит

- Один экран (single page app) с:
  - верхней панелью фильтров;
  - списком/сеткой карточек;
  - базовой пагинацией или кнопкой "Load more".
- Локальная генерация HTML-страницы из данных Supabase.
- Фильтр по `dataset_slug` (поставщик данных/импорт).
- Базовый текстовый поиск по заголовку и текстам промптов.
- Отображение всех prompt-вариантов внутри карточки.

### Не входит

- SEO, SSR/SSG, sitemap, robots, мета-теги.
- Авторизация, роли, админка.
- Запись обратно в БД.
- Генерация изображений.
- Сложные метрики и продуктовая аналитика.

---

## 3. Источник данных (локально)

### 3.1 Формат входа

- Основной источник: Supabase (таблицы `prompt_cards`, `prompt_variants`, `prompt_card_media`).
- Прототип запускается локально, но тянет данные из текущей базы проекта `photo2sticker`.

### 3.2 Требования к загрузке данных

- При старте приложение выполняет загрузку карточек из Supabase.
- На стороне UI данные приводятся к единой структуре `PromptCardViewModel`.
- Если часть полей отсутствует, карточка рендерится с fallback-значениями (`-`, `Нет данных`, пустые массивы).

### 3.3 Минимальная структура `PromptCardViewModel`

- `id` (string, уникальный ключ в пределах загрузки)
- `datasetSlug` (string)
- `sourceMessageId` (string | number)
- `sourceDate` (string | null)
- `title` (string)
- `promptTextsRu` (string[])
- `promptTextsEn` (string[])
- `hashtags` (string[])
- `photoCount` (number)
- `promptCount` (number)
- `warnings` (string[])
- `previewImagePath` (string | null)

## 4. Функциональные требования UI

## 4.1 Layout

- Верхняя панель:
  - select `Dataset` (все уникальные `datasetSlug` + `All`);
  - input `Search` (по `title` и `promptTextsRu`);
  - чекбокс `Only with warnings`;
  - счетчик: `N cards found`.
- Основная зона:
  - responsive grid карточек (desktop-first);
  - 3 колонки на desktop, 2 на tablet, 1 на mobile.

## 4.2 Карточка промта

Каждая карточка отображает:

- мини-превью изображения (если есть `previewImagePath`, иначе placeholder);
- `title`;
- служебную строку: `datasetSlug`, `sourceMessageId`, дата;
- бейджи: `photoCount`, `promptCount`, `warnings`;
- блок `Prompt RU` (все варианты списком);
- блок `Hashtags`;
- кнопку `Copy RU prompts` (копирует все RU промпты через `\n\n`).

## 4.3 Фильтрация и поиск

- Dataset-фильтр применяется мгновенно на клиенте.
- Поиск: `contains` без учета регистра.
- Если включен `Only with warnings`, остаются карточки где `warnings.length > 0`.
- Все фильтры комбинируются через логическое `AND`.

## 4.4 Детальный просмотр

- В MVP детальная модалка не обязательна.
- Опционально: раскрытие карточки кнопкой `Expand`, если текст длинный.

---

## 5. UX/визуальные требования

- Светлая тема, нейтральный минималистичный стиль.
- Фокус на читаемость текста промптов:
  - моноширинный или полумоноширинный блок для prompt-текста;
  - ограничение высоты + скролл внутри блока при длинном тексте.
- Кликабельные элементы имеют видимые hover/focus состояния.
- Карточки не "прыгают" при загрузке изображений (фиксировать контейнер превью по ratio).

---

## 6. Технические требования (MVP)

- Текущая реализация: server-side скрипт на `TypeScript` (`tsx`) + статический `index.html` на vanilla JS/CSS.
- Источник данных: Supabase service role (через локальный скрипт), без прямого чтения `.xlsx`.
- Скрипт запрашивает:
  - `prompt_cards` (базовые поля карточки),
  - `prompt_variants` (тексты промптов),
  - `prompt_card_media` (превью-изображения).
- Для приватного bucket `prompt-images` используются signed URLs с TTL.
- После генерации HTML данные хранятся в памяти страницы (client-side фильтры).
- Ошибки загрузки данных показываются в UI (`Не удалось загрузить данные из БД` + причина).

---

## 7. Состояния экрана

- `loading` — идет загрузка/преобразование данных.
- `ready` — список карточек доступен.
- `empty` — после фильтров 0 карточек.
- `error` — не удалось прочитать данные.

---

## 8. Критерии приемки

MVP считается готовым, если:

1. Приложение локально запускается одной командой и открывается в браузере.
2. Загружает карточки из Supabase (`prompt_cards`, `prompt_variants`, `prompt_card_media`).
3. Фильтр по `datasetSlug` работает корректно.
4. Поиск по заголовку и RU промптам работает корректно.
5. Карточки отображают ключевые поля (title, prompts, badges, warnings).
6. Кнопка `Copy RU prompts` копирует текст в буфер.
7. Empty/Error состояния визуально обработаны.

---

## 9. Команды запуска

- Сборка прототипа:
  - `npm run build:prompt-cards:prototype`
- Только для конкретного датасета:
  - `npm run build:prompt-cards:prototype -- --dataset lexy_15.02.26`
- Ограничить количество карточек:
  - `npm run build:prompt-cards:prototype -- --limit 300`
- Изменить TTL signed URL (часы):
  - `npm run build:prompt-cards:prototype -- --sign-hours 12`

Результат сборки:
- `export/prototype/prompt-cards/index.html`
