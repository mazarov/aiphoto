# Требования: трёхвкладочный UI расширения Steal This Vibe

> **Дата:** 2026-03-27  
> **Статус:** реализовано в коде (см. `extension/sidepanel/stv-core.js`, `stv-prompt-assembly.js`).  
> **Связанные документы:** [extension-ui-spec.md](../extension-ui-spec.md), [steal-this-vibe-extension.md](../steal-this-vibe-extension.md).

---

## 1. Продукт и задача пользователя

**Продукт:** Chrome side panel «Steal This Vibe» / PromptShot — перенос визуального стиля с референса на фото пользователя через извлечение стиля (JSON) и генерацию изображения.

**Задачи пользователя (три зоны внимания):**

| Зона | Цель |
|------|------|
| A — Prompt to photo | Загрузить/выбрать референс, получить распознанный промпт, при необходимости отредактировать его по блокам. |
| B — AI generation | Загрузить своё фото, настроить модель и параметры, запустить генерацию, видеть результат в сетке сравнения. |
| C — My generations | Просматривать историю запусков, экспорт/очистка — без изменения текущей логики. |

---

## 2. Информационная архитектура

Под общим **topbar** (бренд, язык, выход) и **полосой метаданных** (кредиты, статус, пользователь) размещается **горизонтальная навигация из трёх вкладок**:

1. **Prompt to photo** (`prompt`) — извлечение и редактирование промпта по референсу.  
2. **AI generation** (`generate`) — сетка «ваше фото / референс / результат», внешний вид (grooming), настройки генерации, primary CTA.  
3. **My generations** (`history`) — только блок истории запусков (тот же состав, что и ранее внизу shell).

**Персист:** `activeTab` сохраняется в `stv_state_v2` вместе с остальным состоянием панели.

## 3. Вкладка «Prompt to photo»

### 3.1 Layout

- **Слева:** превью **референса** (тот же источник, что и сейчас: URL со страницы или файл с ПК через `+`). Удаление референса — как в текущей логике.  
- **Справа:** блок **итогового промпта** (краткий вид: свёрнутый текст / первые строки + при необходимости scroll).  
- Кнопка **«Редактировать»** переключает режим: вместо одного блока показываются **девять полей** (textarea):

| UI label | Ключ в `state.style` (JSON) |
|----------|-----------------------------|
| Scene | `scene` |
| Genre | `genre` |
| Pose | `pose` (если API вернул `subject_pose`, в UI маппится в то же поле) |
| Lighting | `lighting` |
| Camera | `camera` |
| Mood | `mood` |
| Color | `color` |
| Clothing | `clothing` |
| Composition | `composition` |

- Одна кнопка **«Сохранить»** в режиме редактирования: применяет значения полей к `state.style`, собирает текст **на клиенте** (без повторного `POST /api/vibe/expand`), обновляет тело, уходящее в `POST /api/generate`, и превью «как на сервере».

### 3.2 Извлечение промпта без генерации

Если референс есть, но стиля ещё нет (не было extract): показать **empty state** и кнопку **«Извлечь промпт»** (или эквивалент i18n), выполняющую цепочку **extract → expand** (+ assemble, если продукт снова включит grooming-pause для нелегаси цепочек), **без** вызова генерации изображения.

### 3.3 Карточка JSON / pipeline spec

Блок «Шаг 1: JSON стиля» и кнопка pipeline-spec по возможности размещаются на этой вкладке (например, в `<details>`), чтобы не загромождать вкладку генерации.

### 3.4 Поведение Save (клиентская сборка)

**Решение продукта:** после правки блоков **не** вызывать expand; клиент собирает:

1. Тело из непустых полей в порядке `scene → … → composition` в формате `Label:\n<text>` — зеркало `buildLegacyVibeFullPromptBody` ([`vibe-legacy-prompt-chain.ts`](../../landing/src/lib/vibe-legacy-prompt-chain.ts)).  
2. Добавление блоков grooming по чекбоксам — зеркало `appendLegacyGroomingPolicyBlocks`.  
3. Строка для отображения «полной» сборки — зеркало `assembleVibeFinalPrompt` ([`vibe-gemini-instructions.ts`](../../landing/src/lib/vibe-gemini-instructions.ts)) с флагом `finalPromptAssumesTwoImages` из последнего expand.

**Риск:** при изменении критических правил на сервере нужно синхронизировать копию в расширении (`stv-prompt-assembly.js`).  
**Source of truth для запуска:** в `POST /api/generate` уходит **unprefixed** тело (legacy + grooming); финальная склейка с CRITICAL RULES выполняется в `generate-process`, как сегодня.

## 4. Вкладка «AI generation»

- Секция **«Фото, референс и результат»** — текущая `stv-compare-grid` (ваше фото | референс | результат).  
- Секция **«Внешний вид»** — grooming-чекбоксы и кнопка «Продолжить генерацию», если применимо (логика без изменений).  
- Секция **«Настройки»** — модель, соотношение сторон, размер, температура extract, режим «Свой промпт», большое поле промпта (редактируемое), раскрывающийся превью финальной сборки.  
- **Текущий промпт (кратко):** кликабельный элемент (chip / строка) → переключение на вкладку **Prompt to photo** и фокус на блок промпта.  
- Кнопки **«Сгенерировать»** и **«Купить кредиты»** — **только** на этой вкладке.  
- Блоки **«Дополнительные действия»** и **«Для разработчиков»** — на этой вкладке внизу main-карточки.

---

## 5. Вкладка «My generations»

- Содержимое карточки истории **без изменений**: заголовок, счётчик, экспорт JSON, очистка, список карточек запусков, существующие обработчики.

---

## 6. Визуальный стиль

- Токены `--stv-*` из [`extension/sidepanel/styles.css`](../extension/sidepanel/styles.css).  
- Вкладки: компактный **underline** или **pill** в одну строку; активная вкладка — акцент indigo/violet, не уменьшать область клика ниже 40px по высоте где возможно.  
- Двухколоночный блок на вкладке 1: на очень узкой панели допускается вертикальный stack (референс сверху, промпт снизу).

---

## 7. UX и краевые случаи

- При первом открытии вкладки по умолчанию: **`generate`** (удобно сразу загрузить фото и нажать «Сгенерировать») или **`prompt`** — зафиксировано в коде как `generate`.  
- Нет референса на вкладке 1: подсказка + ссылка на поведение «Steal this vibe» / «+».  
- Нет стиля: кнопка извлечения; после успешного extract+expand поля заполняются из `state.style`.  
- Совместимость с **scheduleAssemblePrompt / grooming:** для legacy-цепочки assemble на сервере отключён; смена галочек инициирует **runExpand** с текущим `state.style`. После локального Save `state.style` уже обновлён — expand подхватит актуальные поля.
