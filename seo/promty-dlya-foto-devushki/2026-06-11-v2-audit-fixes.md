# /promty-dlya-foto-devushki/ — итерация v2 (аудит качества)

## Найденные проблемы и фиксы

### P0-1. Canonical с трейлинг-слешем при 308-редиректе (сайтвайд)

**Проблема.** Прод: `/promty-dlya-foto-devushki/` → 308 → `/promty-dlya-foto-devushki` (Next.js default, `trailingSlash` не задан). При этом `<link rel="canonical">` указывал на версию **со** слешем, а sitemap — **без**. Конфликт сигналов: canonical ведёт на URL, который отдаёт редирект.

**Фикс.** `landing/src/lib/route-resolver.ts`: хелпер `stripTrailingSlash`, применён к `canonicalPath` (L1/L2/L3) и `parentPath`. Canonical, sitemap и фактический URL теперь совпадают (без слеша). Затрагивает **все** programmatic-страницы, не только devushka.

### P0-2. Нет точного вхождения «промты для девушки» (топ-запрос ядра)

**Проблема.** Самая частотная фраза ядра отсутствовала на странице в точной форме.

**Фикс.** Intro начинается с «Готовые промты для девушки: …». Заодно убрана спам-форма «промт ии девушка работает без доработки» → «ИИ-промт для фото девушки работает без доработки» (риск Баден-Бадена).

### P1-3. Тонкий контент (~550 слов текста)

**Фикс.** Новое поле `seoTextBlocks?: { h2; paragraphs[] }[]` в `SeoContent`, рендер после FAQ в `[...slug]/page.tsx`. Для devushka — блок **«Как составить промт для фотосессии девушки»**, 3 абзаца (~1 400 знаков): структура промта (внешность → одежда → поза → фон → свет), естественные вхождения «нейрофотосессия», «селфи», «на море», «деловой», «реалистичное фото». Закрывает СЧ-группы и «как»-интент без новых L2. Итог: ~750 слов RU-текста.

### P1-4. Иллюстрация «День рождения» не резолвилась (3/4)

**Проблема.** `titleIncludes: "день рожд"` не находил карточку в кластере devushka.

**Фикс.** Прямой `cardSlug: "devushka-v-zolotom-vechernem-plate-s-tortom-na-prazdnike-pri-vspyshke-fotokamery-c2c9b"` + fallback `titleIncludes: "торт"`. Теперь 4/4.

### P1-5. Генерик-H2 «Как использовать промт»

**Фикс.** Новое поле `howToTitle?: string` в `SeoContent`; для devushka — «Как использовать промт для фото девушки» (вхождение ключа в H2).
